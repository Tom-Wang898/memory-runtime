# v0.1.1-alpha

`memory-runtime` now handles ambiguous short references more defensively.

This patch release focuses on one class of failure:

- short prompts like `route A`, `option B`, `this`, or `that`
- wrapper-generated checkpoint noise polluting hot memory
- cold recall overreaching when the query is too small to disambiguate safely

## What changed

- added scoped cold-recall anchoring for ambiguous short queries
- added explicit `recallQueryStrategy` diagnostics:
  - `none`
  - `direct`
  - `anchored`
  - `suppressed`
- wrappers no longer write synthetic automatic checkpoint summaries into hot memory
- hot-memory summary resolution now ignores synthetic fallback summaries when choosing anchors
- added regression coverage for:
  - anchored short-reference recall
  - suppression when no anchor exists
  - summary sanitization behavior
- removed a local absolute filesystem path from the social preview compose asset

## Why this release exists

The runtime already preserved raw user prompts and stayed fail-open, but it still
had a gap around short-reference ambiguity.

If cold recall was queried with only the current short phrase, retrieval could
pull durable facts from the wrong topic scope.

This release tightens that behavior:

- if hot memory provides a recent project-local anchor, cold recall uses it
- if no safe anchor exists, cold recall is intentionally skipped
- the runtime now prefers less recall over wrong recall

## Behavior change

Before:

- ambiguous short queries could still reach cold memory directly
- wrapper-generated summaries could become accidental retrieval anchors

After:

- ambiguous short queries are either anchor-expanded or suppressed
- wrapper checkpoints only persist the active task, not synthetic summary text
- callers can inspect `diagnostics.recallQueryStrategy` to understand the recall path

## Migration notes

- no schema migration is required
- no user config change is required
- existing wrapper installs keep working after pulling the latest code
- already-running shells still need a reload if the wrapper scripts changed

## Verification used for this release

- `npm run check:all`
- `npm run bench:bootstrap`
- `npm run bench:tokens`

Latest observed benchmark outputs during release prep:

- bootstrap benchmark: `avgMs = 0.165`, `maxMs = 0.711` over 20 iterations
- token benchmark: `compactTokens = 65`, `naiveTokens = 375`, `estimatedSavings = 310`

## Feedback focus

The most useful feedback for this patch:

- whether ambiguous project shorthand now stays correctly scoped
- whether any legitimate cold recall is being suppressed too aggressively
- whether wrapper checkpointing remains helpful without synthetic summaries
- whether the new diagnostics are enough to debug recall behavior in the field

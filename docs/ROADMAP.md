# Roadmap

## Phase 0: Scaffold

- establish repo layout
- define contracts
- define provider interfaces
- define Codex bootstrap surface

## Phase 1: Local hot provider

- add SQLite schema
- implement project capsule read/write
- implement checkpoint updates
- add latency-focused tests

## Phase 2: Memory Palace adapter

- implement gist/fact recall
- implement promotion path
- add timeout and fail-open behavior

## Phase 3: Codex host integration

- add wrapper command
- add bootstrap envelope rendering
- add checkpoint emission flow

## Phase 4: Evaluation

- measure bootstrap latency
- measure token savings
- verify no fidelity regression on implementation tasks

## Phase 5: Skill governance

- add public skill governance with audit, explicit apply, rollback, and benchmark flows
- detect token-heavy and host-coupled local skills
- keep duplicate resolution in manual-review space until managed scopes are stronger

## Phase 6: Future governance hardening

- widen managed transforms beyond the current safe subset
- add duplicate-skill resolution workflows with explicit user choice
- add richer provider-specific benchmark fixtures
